# Email delivery goes through the worker queue, templates in a shared package

Transactional email (starting with Organization invitations in LEA-11) is never sent inline by `apps/api`. The API enqueues a job on the `email` BullMQ queue; `apps/worker` renders the template and sends via Resend with retry and backoff, because the worker owns outbound delivery and async side effects in the approved architecture. The naive alternative - fire-and-forget in the API request flow, as the midday reference does - loses emails on provider failure and contradicts the spec's app ownership. Email templates and the Resend client live in `packages/email` although the worker is the only initial consumer: templates are presentational, boring, and shared by nature (like UI primitives), and more senders (auth flows, notifications) are expected.
